üìñ How SQL Injection Works

SQL injection works because the application uses string concatenation to build SQL queries, rather than parameterized queries. This allows attackers to manipulate the SQL logic.

The Vulnerability:

query = f"SELECT * FROM users WHERE username='{username}' AND password='{password}'"

User input is directly inserted into the query string without sanitization.

Example Attack: ' OR '1'='1

If an attacker enters ' OR '1'='1 as the password, the query becomes:

SELECT * FROM users WHERE username='admin' AND password='' OR '1'='1'

The single quote ' closes the password string early, then OR '1'='1' adds a condition that's always TRUE.

Why It Succeeds:

SQL evaluates: (username='admin' AND password='') OR ('1'='1')

Since '1'='1' is always true, the entire WHERE clause returns true, giving access to user accounts!

üß™ Other Payloads to Try

Go back and try these other SQL injection techniques:

Username field bypass: admin'--

The -- comments out everything after (including password check). Enter anything as password.

Always true (numeric): ' OR 1=1--

Uses numeric comparison instead of string. Works in both username or password field.

Close and comment: admin' OR '1'='1'--

Combines username injection with always-true condition.

OR with different syntax: ' OR 'a'='a

Same concept - any comparison that's always true works!

UNION attack (advanced): ' UNION SELECT 4, 'hacked', 'password'--

Returns a fake user row. Shows how attackers can extract data.

Extract all passwords (data exfiltration): ' UNION SELECT id, username, password FROM users--

Dumps ALL usernames and passwords from the database! This shows how SQL injection can lead to massive data breaches.
